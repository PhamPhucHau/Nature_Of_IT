================================================================================
                              TECHNICAL DETAILS
================================================================================

Root Cause:
  The conversations state was being set without guaranteeing it was an array.
  If the API response was delayed, undefined, or in an error state, the
  component would try to call .map() on a non-array, causing the crash.

Solution Pattern:
  Use defensive programming with Array.isArray() checks and fallbacks.
  This ensures components always receive the data type they expect.

Example Pattern:
  const data = await api.call()
  const safe = Array.isArray(data) ? data : []
  setState(safe)

Benefits:
  - No runtime errors
  - Better error messages
  - Graceful degradation
  - Improved UX
