Welcome to Phase 3: SQL Parser Implementation!

This project implements a complete SQL parser that converts SQL strings into

lexical analysis, syntax analysis, and AST construction.

================================================================================
                            SQL GRAMMAR
================================================================================

The parser supports:

SELECT column_list FROM table_name [WHERE condition]
INSERT INTO table_name (column_list) VALUES (value_list)
CREATE TABLE table_name (column_def, ...)
DELETE FROM table_name [WHERE condition]

Examples:

SELECT * FROM users
SELECT id, name FROM users WHERE id = 5
INSERT INTO users (id, name) VALUES (1, 'Alice')
CREATE TABLE users (id INT, name VARCHAR(50))
DELETE FROM users WHERE id = 5

================================================================================
                            KEY CONCEPTS
================================================================================

Lexical Analysis:
  Input:  "SELECT * FROM users"
  Output: [SELECT] [*] [FROM] [users] [EOF]

Syntax Analysis:
  Tokens: [SELECT] [*] [FROM] [users]
  AST:    SelectStatement(table="users", columns=["*"])

Abstract Syntax Tree:
  SelectStatement
  ├── table_name: "users"
  ├── columns: [Identifier("*")]
  └── where_clause: NULL
