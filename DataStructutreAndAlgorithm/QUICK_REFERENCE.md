ArrayList.add() is O(1) not O(n)?\n\n```\nCapacity: 10  |  20  |  40  |  80  |  160\nResizes:   1  +  1   +  1   +  1   +   1   = 5 resizes\n                                            for 160 adds\n\nResizes resize  n items:\n10вҶ’20: 10 copies\n20вҶ’40: 20 copies\n40вҶ’80: 40 copies\n80вҶ’160: 80 copies\nTotal: 150 copies\n\nAverage per add: 150 / 160 вүҲ 1 operation (O(1) amortized!)\n\nKey insight: Resizes double capacity, not +1\nвҶ’ Sum = 1 + 2 + 4 + 8 + ... + n = 2n = O(n) total\nвҶ’ O(n) total / n operations = O(1) average per operation\n```\n\n---\n\n## рҹҢҗ Network Software Patterns\n\n### Pattern 1: Fast Lookup\n```\nProblem: Find connection by source IP\nSolution: Hash Table<String srcIP, Connection>\nTime: O(1) average\n```\n\n### Pattern 2: Priority Processing\n```\nProblem: Process high-priority packets first\nSolution: Priority Queue (Heap)\nTime: O(log n) per packet\n```\n\n### Pattern 3: Ordered Iteration\n```\nProblem: Process connections in creation order\nSolution: LinkedList or Deque\nTime: O(1) at ends, O(n) random access\n```\n\n### Pattern 4: Batch Processing\n```\nProblem: Process multiple items efficiently\nSolution: Ring Buffer / Circular Queue\nTime: O(1) enqueue/dequeue, reuse memory\n```\n\n### Pattern 5: Shortest Path\n```\nProblem: Find route with minimum cost\nSolution: Dijkstra's algorithm\nTime: O((V+E) log V) with heap\n```\n\n---\n\n## рҹ“қ Code Templates\n\n### C: Array Access\n```c\nint arr[10] = {1, 2, 3, 4, 5};\nint val = arr[2];        // O(1) access\nint len = sizeof(arr) / sizeof(arr[0]);  // Get Mistakes\n\n### 1. Binary Search on Unsorted Array\n```\nвқҢ WRONG:\nint[] unsorted = {5, 2, 8, 1, 9};\nbinarySearch(unsorted, 8);  // May not find it!\n\nвң… CORRECT:\nArrays.sort(unsorted);\nbinarySearch(unsorted, 8);  // Now works\n```\n\n### 2. Forgetting to Check NULL\n```\nвқҢ WRONG:\nNode current = head;\ncurrent.next = newNode;  // NPE if head is null!\n\nвң… CORRECT:\nif (head == null) {\n    head = newNode;\n} else {\n    Node current = head;\n    while (current.next != null) {\n        current = current.next;\n    }\n    current.next = newNode;\n}\n```\n\n### 3. Off-by-One in Loops\n```\nвқҢ WRONG:\nfor (int i = 0; i <= arr.length; i++) {  // ArrayIndexOutOfBounds!\n    print(arr[i]);\n}\n\nвң… CORRECT:\nfor (int i = 0; i < arr.length; i++) {   // Correct\n    print(arr[i]);\n}\n```\n\n### 4. Not Freeing Memory (C)\n```\nвқҢ WRONG:\nvoid createNode() {\n    Node* node = (Node*)malloc(sizeof(Node));\n    // ... use node ...\n    // forgot to free!\n}\n\nвң… CORRECT:\nvoid createNode() {\n    Node* node = (Node*)malloc(sizeof(Node));\n    // ... use node ...\n    free(node);  // Clean up\n}\n```\n\n---\n\n## рҹҺ“ 30-Minute Review\n\n### 5 Minutes: Arrays\n- O(1) random access\n- O(n) insert/delete (except end)\n- Use when: need fast lookup\n\n### 5 Minutes: Linked Lists\n- O(1) insert/delete at ends\n- O(n) random access\n- Use when: frequent add/remove at head/tail\n\n### 5 Minutes: Hash Tables\n- O(1) average lookup by key\n- O(n) worst case (collisions)\n- Use when: need key-value storage\n\n### 5 Minutes: Heaps\n- O(log n) insert/remove\n- Find min/max in O(1)\n- Use when: need priority queue\n\n### 5 Minutes: Algorithms\n- Binary Search: O(log n) on sorted data\n- Quick Sort: O(n log n) average\n- Dijkstra: O((V+E) log V) shortest path\n\n---\n\n## рҹҡҖ Next Steps\n\n1. **Understand**: Read the code and stories\n2. **Compile**: Get the code running\n3. **Modify**: Change test cases and see what happens\n4. **Practice**: Solve problems using these structures\n5. **Internalize**: Recall without looking at code\n\n---\n\nGood luck! рҹҺҜ\n
