# Data Structures & Algorithms Implementation Guide\n\n## 📖 How to Use This Repository\n\n---\n\n## 🎯 Part 1: Data Structures (Cấu Trúc Dữ Liệu)\n\n### What is a Data Structure?\n\n**Cấu trúc dữ liệu là cách tổ chức dữ liệu để truy cập và sửa đổi hiệu quả.**\n\nMỗi cấu trúc dữ liệu là sự cân bằng giữa:\n- ⏱️ **Thời gian** (Time): Nhanh hay chậm?\n- 💾 **Bộ nhớ** (Space): Dùng bao nhiêu bộ nhớ?\n- 🔧 **Độ phức tạp** (Complexity): Dễ hay khó để code?\n\n---\n\n## 📚 Implemented Data Structures\n\n### 1. **Array / ArrayList** (01_Array_ArrayList.*)\n\n#### Story\n```\nBạn có 100 học sinh, cần lưu điểm thi của mỗi người.\nThay vì tạo 100 biến (score1, score2, ...), bạn tạo 1 mảng[100].\n```\n\n#### Key Points\n```\nArray:      [10, 20, 30, 40, 50]\n             ↑   ↑   ↑   ↑   ↑\n             0   1   2   3   4  (index)\n\nAccess tại index i: arr[i] → O(1) (trực tiếp)\n```\n\n#### Time Complexity\n| Operation | Time | Reason |\n|-----------|------|--------|\n| Get | O(1) | Tính địa chỉ: base + index*size |\n| Add (cuối) | O(1)* | Amortized (resize O(n) nhưng hiếm) |\n| Add (đầu) | O(n) | Phải dịch tất cả phần tử |\n| Remove (k) | O(n) | Phải dịch phần tử sau vị trí k |\n\n*Amortized: Trung bình O(1) khi tính cả chi phí resize\n\n#### Network Application\n```\n1. Packet Buffer: Lưu các packet đến tại NIC\n   [Packet1] [Packet2] [Packet3] [...]\n\n2. Socket Table: Danh sách các connection hiện tại\n   [Socket1] [Socket2] [Socket3] [...]\n\n3. Routing Cache: Lưu kết quả lookup gần đây\n```\n\n#### When to Use\n✅ Dùng khi:\n- Cần truy cập ngẫu nhiên (random access)\n- Kích thước đã biết trước\n- Thêm/xóa chủ yếu ở cuối\n\n❌ Không dùng khi:\n- Thường xuyên thêm/xóa ở đầu\n- Kích thước thay đổi liên tục\n\n---\n\n### 2. **Linked List** (02_LinkedList.*)\n\n#### Story\n```\nBạn có một sợi dây và dán các nốt (node) lên.\nMỗi nốt có:\n  - Dữ liệu (data)\n  - Con trỏ tới nốt tiếp theo (next)\n\n[Node1] → [Node2] → [Node3] → NULL\n  |         |         |\n  10        20        30\n```\n\n#### Key Points\n```\nLinked List: 10 → 20 → 30 → NULL\n             ↑              ↑\n           head           tail\n```\n\n#### Time Complexity\n| Operation | Time | Reason |\n|-----------|------|--------|\n| Get (index i) | O(n) | Phải đi từ đầu đến i |\n| Add (đầu) | O(1) | Chỉ cần thay đổi head con trỏ |\n| Add (cuối) | O(n) | Phải tìm tail trước |\n| Remove (đầu) | O(1) | head = head.next |\n| Remove (k) | O(n) | Phải tìm node trước k |\n\n#### Network Application\n```\n1. TCP Connection Queue:\n   Khi client kết nối → addFirst() O(1)\n   Khi xử lý xong → removeFirst() O(1)\n\n2. Packet Queue tại Network Interface:\n   Gói tin đến → addLast()\n   Gói tin xử lý → removeFirst()\n\n3. Message Queue (RabbitMQ, Kafka):\n   Producer: add phần tử vào cuối\n   Consumer: remove phần tử từ đầu\n```\n\n#### When to Use\n✅ Dùng khi:\n- Thường xuyên thêm/xóa ở đầu hoặc cuối\n- Không cần truy cập ngẫu nhiên\n- Kích thước không xác định trước\n\n❌ Không dùng khi:\n- Cần truy cập ngẫu nhiên nhiều\n- Muốn cache locality tốt\n\n---\n\n### 3. **Stack & Queue** (03_Stack_Queue.*)\n\n#### Stack (LIFO - Last In, First Out)\n```\nVí dụ: Chồng đĩa\n\n    ↓ push\n  ┌─────┐\n  │  3  │ ← pop\n  │  2  │\n  │  1  │\n  └─────┘\n\nCampaign:    push 1 → push 2 → push 3 → pop → pop\nResult:      [1] → [1,2] → [1,2,3] → [1,2] → [1]\n```\n\n#### Queue (FIFO - First In, First Out)\n```\nVí dụ: Hàng chờ\n\nEnqueue    Dequeue\n   ↓          ↓\n[1, 2, 3, 4, 5]\n\nCampaign:    add 1 → add 2 → add 3 → remove → remove\nResult:      [1] → [1,2] → [1,2,3] → [2,3] → [3]\n```\n\n#### Time Complexity\n| Operation | Stack | Queue |\n|-----------|-------|-------|\n| Push/Enqueue | O(1) | O(1) |\n| Pop/Dequeue | O(1) | O(1) |\n| Peek | O(1) | O(1) |\n\n#### Network Application\n```\nSTACK:\n- HTTP/HTTPS protocol parsing (match brackets)\n- DFS routing algorithm\n- Expression evaluation in parsers\n\nQUEUE:\n- Packet buffer queue at NIC (ring buffer)\n- TCP SYN queue (listen backlog)\n- Message queue (RabbitMQ, Kafka)\n- Load balancer request queue\n- Scheduler queue in OS kernel\n```\n\n---\n\n## 🔄 Part 2: Algorithms (Thuật Toán)\n\n### What is an Algorithm?\n\n**Thuật toán là một tập hợp các bước để giải quyết một vấn đề.**\n\nMục tiêu:\n- 🎯 **Chính xác** (Correctness): Cho kết quả đúng\n- ⚡ **Nhanh** (Efficiency): Dùng ít thời gian\n- 💾 **Tiết kiệm** (Optimization): Dùng ít bộ nhớ\n\n---\n\n## 🔍 Implemented Algorithms\n\n### 1. **Binary Search** (01_Binary_Search.*)\n\n#### Story\n```\n\"Để tìm từ A đến Z trong từ điển:\nThay vì tìm từ đầu đến cuối (A, B, C, ...),\nhãy mở giữa. Nếu cần chữ trước, loại bỏ nửa sau.\nNếu cần chữ sau, loại bỏ nửa trước.\"\n\nĐó là Binary Search.\n```\n\n#### How it Works\n```\nArray: [1, 3, 5, 7, 9, 11, 13, 15]\nTìm: 7\n\nBước 1: left=0, right=7, mid=3 → arr[3]=7 ✓ FOUND\n\nNếu tìm 6:\nBước 1: left=0, right=7, mid=3 → arr[3]=7 > 6 → right=2\nBước 2: left=0, right=2, mid=1 → arr[1]=3 < 6 → left=2\nBước 3: left=2, right=2, mid=2 → arr[2]=5 < 6 → left=3\nBước 4: left=3, right=2 → INVALID, NOT FOUND\n```\n\n#### Time Complexity\n```\nLinear Search:  O(n) → 1 triệu phần tử = 500,000 lần check (trung bình)\nBinary Search:  O(log n) → 1 triệu phần tử = 20 lần check\n\n→ Binary Search nhanh 50,000 lần! ⚡\n```\n\n#### Requirements\n⚠️ **IMPORTANT**: Mảng PHẢI đã sắp xếp!\n\n```java\n// ✓ Đúng\nint[] sorted = {1, 3, 5, 7, 9};\nbinarySearch(sorted, 5);\n\n// ✗ Sai\nint[] unsorted = {5, 1, 9, 3, 7};\nbinarySearch(unsorted, 5);  // Có thể không tìm thấy!\n```\n\n#### Network Application\n```\n1. DNS Cache Lookup\n   domains = [amazon.com, facebook.com, google.com, microsoft.com]\n   IPs =     [54.239..., 31.13..., 142.250..., 13.107...]\n   \n   Lookup \"google.com\" → binary search → O(log n)\n\n2. Routing Table Search\n   Tìm route có IP destination gần nhất\n   \n3. Bandwidth Allocation\n   Tìm bandwidth slot phù hợp\n```\n\n---\n\n### 2. **Sorting: Quick Sort & Merge Sort** (02_Sorting_QuickSort_MergeSort.*)\n\n#### Quick Sort\n\n**Story**\n```\n\"Chọn đại một phần tử (Pivot), chia danh sách làm 2:\n- Phần tử < Pivot → trái\n- Phần tử > Pivot → phải\nLặp lại với từng nửa.\"\n```\n\n**How it Works**\n```\nArray: [64, 34, 25, 12, 22, 11, 90, 88, 45, 50]\nPivot = 50\n\nBước 1: Partition\n[34, 25, 12, 22, 11, 45] [50] [64, 90, 88]\n          <50                    >50\n\nBước 2: Recursive sort từng phần\n```\n\n**Time Complexity**\n```\nBest Case:    O(n log n) - Pivot chia đôi mỗi lần\nAverage Case: O(n log n) - Thường tốt\nWorst Case:   O(n²)      - Pivot luôn là min/max (hiếm)\n\nSpace:        O(log n)   - Call stack depth\n```\n\n#### Merge Sort\n\n**Story**\n```\n\"Chia mảng làm đôi cho đến khi có 1 phần tử.\nSau đó trộn 2 mảng con sắp xếp lại theo cách khôn ngoan.\"\n```\n\n**How it Works**\n```\nArray: [64, 34, 25, 12]\n\nBước 1: Chia\n[64, 34]      [25, 12]\n  |             |\n[64] [34]    [25] [12]\n\nBước 2: Trộn\n[34, 64]      [12, 25]\n    |             |\n  [12, 25, 34, 64]\n```\n\n**Time Complexity**\n```\nBest Case:    O(n log n)\nAverage Case: O(n log n)\nWorst Case:   O(n log n) ← Đảm bảo!\n\nSpace:        O(n)  - Temporary arrays\nStable:       Yes   - Giữ thứ tự phần tử bằng nhau\n```\n\n#### Quick Sort vs Merge Sort\n\n| Aspect | Quick Sort | Merge Sort |\n|--------|-----------|------------|\n| Average Time | O(n log n) | O(n log n) |\n| Worst Time | O(n²) | O(n log n) |\n| Space | O(log n) | O(n) |\n| In-place | Yes | No |\n| Stable | No | Yes |\n| Cache Friendly | Yes | No |\n| Best for | General use | Need guarantee |\n\n#### Network Application\n```\n1. QoS Packet Scheduling\n   Sort packets by priority\n   Merge Sort if need stable (FIFO within same priority)\n\n2. Routing Metrics\n   Sort routes by cost\n   Quick Sort (in-place, usually faster)\n\n3. Bandwidth Allocation\n   Sort flows by allocated bandwidth\n\n4. Log Analysis\n   Sort network logs by timestamp/severity\n```\n\n---\n\n## 🌐 Network Software Architecture\n\n### Example: High-Performance Packet Processing\n\n```\nIncoming Packet\n        ↓\n   [Ring Buffer]         ← Circular Queue (high speed)\n     ↓\n  [Hash Table]           ← Connection lookup (O(1))\n     ↓\n[Priority Queue]         ← QoS scheduling (Heap)\n     ↓\n  [Thread Pool]          ← Process packet\n     ↓\n   [Ring Buffer]         ← Output\n     ↓\n   Network Card\n```\n\nEach layer uses optimal data structure:\n- **Ring Buffer**: O(1) circular reuse, no malloc/free\n- **Hash Table**: O(1) lookup\n- **Priority Queue**: O(log n) insert/remove, maintains order\n- **Thread Pool**: Queue of work items\n\n---\n\n## 🚀 Quick Start\n\n### C Compilation & Execution\n\n```bash\n# Navigate to directory\ncd DataStructure/\n\n# Compile\ngcc -o demo 01_Array_ArrayList.c\n\n# Run\n./demo\n\n# Example output:\n# ✓ Tạo Array xong.        → Hash Table\nNeed frequent additions? → Linked List\nNeed both?               → Trade-off or hybrid solution\n```\n\n### 2. **Understand Trade-offs**\n```\nFaster access    ← → Easier insertion\nSmall memory     ← → Quick operations\nSimple code      ← → High performance\n```\n\n### 3. **Think in Layers**\n```\nApplication Logic\n    ↓\n   Algorithm (Sorting, Searching, Path Finding)\n    ↓\n   Data Structure (Array, Hash Table, Heap)\n    ↓\n   Hardware (CPU, Memory, Cache)\n```\n\n### 4. **Amortized Analysis Matters**\n```\nArrayList.add() seems O(n) (resize)\nBut O(1) amortized because resize rare\n\nThis is why ArrayList is practical despite theoretical O(n)\n```\n\n---\n\n## 🎓 Learning Strategy\n\n### Week 1-2: Foundation\n- [ ] Understand Array vs Linked List trade-offs\n- [ ] Implement Stack and Queue\n- [ ] Practice basic operations\n\n### Week 3-4: Searching & Sorting\n- [ ] Master Binary Search\n- [ ] Understand partition concept (Quick Sort)\n- [ ] Compare sorting algorithms\n\n### Week 5-6: Advanced\n- [ ] Hash Tables and collision handling\n- [ ] Trees and traversal\n- [ ] Graphs and shortest path\n\n### Week 7-8: Application\n- [ ] Solve coding problems\n- [ ] Analyze real-world systems\n- [ ] Design solutions for constraints\n\n---\n\n## 📚 Further Reading\n\n- \"Introduction to Algorithms\" (CLRS) - Gold standard textbook\n- \"Algorithms\" by Sedgewick & Wayne - Clear explanations\n- LeetCode - Practice problems\n- VisuAlgo - Algorithm visualization\n\n---\n\nHappy learning! 🚀\n
